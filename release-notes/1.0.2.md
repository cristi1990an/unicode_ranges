# unicode_ranges v1.0.2

Draft notes for the next patch release.

## Changed

- Optimized owning-string insert paths to avoid temporary transcoding buffers for repeated character inserts and cross-encoding `insert_range(...)` overloads. UTF-8, UTF-16, and UTF-32 owning strings now pre-measure the inserted range, grow the destination once, move the existing tail once, and write directly into the insertion gap where the source encoding cannot alias the destination buffer.
- Rvalue owning strings now return move-only owning views from `chars()`, `reversed_chars()`, `char_indices()`, `grapheme_indices()`, `graphemes()`, and the split-family view methods, avoiding dangling views when a view is kept from a temporary owning string. String-view overloads remain borrowed views.
- Optimized owning-string materialization from same-encoding `chars()` and rvalue `reversed_chars()` views: `std::from_range`, `append_range`, `assign_range`, and `insert_range` now use direct same-encoding storage paths and can reuse storage from rvalue owning views when allocator compatibility allows it.
