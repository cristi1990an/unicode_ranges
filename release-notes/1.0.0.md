# unicode_ranges v1.0.0

## Summary

`unicode_ranges` `v1.0.0` is the first stable `1.x` source release.

This release establishes the supported public API and build surface for `1.x`:

- validated UTF-8, UTF-16, and UTF-32 character, view, and owning-string types
- a compiled library target with first-party CMake build/install/export support
- vendored pinned `simdutf` as the runtime backend for hot UTF validation and UTF-8 transcoding paths
- optional ICU-backed locale-aware casing
- a documented `1.x` stability contract in `STABILITY.md`

## Highlights

### Stable public entry points

The primary umbrella headers for `1.x` are:

- `unicode_ranges_borrowed.hpp`
- `unicode_ranges_all.hpp`

Legacy wrappers:

- `unicode_ranges.hpp`
- `unicode_ranges_full.hpp`

are still shipped for older code, but the two headers above are the intended stable names.

### Compiled-library model

The supported integration model is now a compiled library target:

- CMake target: `unicode_ranges::unicode_ranges`
- installed package consumption via `find_package(unicode_ranges CONFIG REQUIRED)`

Consumers should build and link the library target rather than compiling `unicode_ranges.cpp` directly into every downstream target.

### Runtime backend

Runtime UTF-8 validation and runtime UTF-8 <-> UTF-16/UTF-32 transcoding now use vendored pinned `simdutf` `v7.7.0`.

That keeps the validated `unicode_ranges` type model and error surface while using a proven high-performance backend for the hot UTF boundary operations.

### UTF-16 / UTF-32 expansion

The library now ships first-class UTF-16 and UTF-32 support alongside UTF-8, including:

- character types
- borrowed views
- owning strings
- literals
- conversions
- normalization
- casing
- grapheme-aware operations

### Locale-aware casing and comparisons

When ICU is enabled, the library provides:

- locale-aware lower/upper/title/case-fold transforms
- locale-aware ignore-case comparisons
- locale token support through `locale_id` and `_locale`

## Support Policy

`v1.0.0` defines a `1.x` source-compatibility contract, not a universal ABI-compatibility promise.

See:

- `STABILITY.md`
- `README.md`
- `CHANGELOG.md`

## Release Model

This release is source-first:

- Git tag
- GitHub source archives
- first-party CMake install/export package from source

Prebuilt binaries are not part of the `1.x` release contract today.
