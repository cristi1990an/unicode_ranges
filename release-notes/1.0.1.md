# unicode_ranges v1.0.1

## Summary

`unicode_ranges` `v1.0.1` is a patch release that fixes release-readiness issues found immediately after `v1.0.0`.

## Fixes

- Fixed direct inclusion of `unicode_ranges/unicode_tables.hpp`.
- Gave `unicode_ranges/unicode_tables_constexpr.hpp` its own include guard so the compatibility wrapper no longer suppresses the generated table header.
- Made the generated constexpr table header self-contained for its standard-library dependencies.
- Added regression coverage for including `unicode_ranges/unicode_tables.hpp` before the umbrella headers.
- Corrected the README zero-copy validation example to use `std::u8string` with `utf8_string_view::from_bytes(...)`.
- Replaced the reserved legacy include guard in `unicode_ranges.hpp`.

## Documentation

- Clarified the libstdc++ 14 range-formatting limitation for helper views in the README quick start.
- Aligned the docs scope language with optional ICU-backed locale-aware casing support.
- Added `unicode_ranges::views` and `unicode_ranges::characters` to the documented `1.x` stable namespace contract.

## Compatibility

No public API removals or behavioral changes are intended. This release is source-compatible with `v1.0.0`.
